## 二、第一阶段：Protégé 本体构建（概念层）

> 答辩重点：证明采用"自上而下（Top-down）"的本体工程方法，而非直接写脚本塞数据。

### 2.1 安装与准备

1. 下载 [Protégé 5.6](https://protege.stanford.edu/)（Desktop 版）
2. 新建 OWL 本体，命名空间建议：`http://example.org/songci#`
3. 前缀：`songci:`

### 2.2 类（Class）设计

在 **Entities** 面板中创建以下核心类及层级关系：

```
Thing
├── Agent（主体）
│   └── Poet（词人）
├── Work（作品）
│   └── Cipai（词牌）
├── Artifact（器物）
│   └── MusicInstrument（乐器）
├── TemporalEntity（时间实体）
│   └── Period（朝代/时期）
└── Concept（概念）
    └── CiStyle（词风/流派）
```

**说明：**

- `Agent` 作为人物的上位类，便于后续扩展（如乐工、评论家）
- `Cipai` 归入 `Work`，强调其作为"词体格式"而非单篇作品
- `Period` 用于时间轴过滤（唐五代、北宋、南宋、宋末）

### 2.3 对象属性（Object Properties）

| 属性名 | Domain | Range | 含义 | 示例 |
|--------|--------|-------|------|------|
| `composed_by` | Cipai | Poet | 词牌由某词人代表 | 念奴娇 → 苏轼 |
| `has_alias` | Cipai | xsd:string | 词牌别称（数据属性） | 念奴娇 → 百字令 |
| `accompanied_by_instrument` | Cipai | MusicInstrument | 词牌常用伴奏乐器 | 慢词 → 笙 |
| `belongs_to_style` | Poet | CiStyle | 词人所属流派 | 姜夔 → 清雅派 |
| `active_in` | Poet | Period | 词人活跃时期 | 姜夔 → 南宋 |
| `classified_as` | Cipai | xsd:string | 词体类型（慢词/小令等） | 念奴娇 → 慢词 |

**注意：** `has_alias`、`classified_as` 为数据属性（Data Properties），在 Protégé 的 **Data properties** 面板单独创建。

### 3.2 图数据库 Schema（节点与关系）

**节点标签：**

| 标签 | 主要属性 | 来源 |
|------|----------|------|
| `:Poet` | `name`, `style`, `period` | CSV + Word |
| `:Cipai` | `name`, `alias`, `type`, `famous_line` | CSV |
| `:Instrument` | `name`, `description` | Word |
| `:Period` | `name`, `start_year`, `end_year` | 手工维护 |

**关系类型：**

| 关系 | 起止 | 含义 |
|------|------|------|
| `REPRESENT` | Poet → Cipai | 代表词人 |
| `ACCOMPANIED_BY` | Cipai → Instrument | 伴奏乐器 |
| `ACTIVE_IN` | Poet → Period | 活跃时期 |
| `BELONGS_TO` | Poet → CiStyle | 所属流派 |
| `ALIAS_OF` | Cipai → Cipai | 别称关系（可选） |

### 3.3 CSV 导入脚本（`scripts/import_csv.py`）

数据文件使用 `data/cipai_all.csv`（UTF-8 BOM 编码）。`代表词人` 字段含多人（以 `、` 分隔），需拆分后逐人建关系。

```python
import pandas as pd
from neo4j import GraphDatabase
from pathlib import Path

URI = "bolt://localhost:7687"
AUTH = ("neo4j", "your_password")
CSV_PATH = Path(__file__).resolve().parent.parent / "data" / "cipai_all.csv"

def import_cipai_csv(driver, csv_path):
    df = pd.read_csv(csv_path, encoding="utf-8-sig")
    with driver.session() as session:
        for _, row in df.iterrows():
            session.run("""
                MERGE (c:Cipai {name: $name})
                SET c.alias = $alias,
                    c.famous_line = $line,
                    c.classic_work = $classic,
                    c.type = $type
            """, name=row["词牌"], alias=row.get("别称", ""),
                 line=row.get("名句", ""), classic=row.get("经典篇目", ""),
                 type=row.get("词体", ""))
            poets = str(row["代表词人"]).split("、")
            for poet in poets:
                poet = poet.strip()
                if not poet:
                    continue
                session.run("""
                    MERGE (p:Poet {name: $poet})
                    WITH p
                    MATCH (c:Cipai {name: $name})
                    MERGE (p)-[:REPRESENT]->(c)
                """, poet=poet, name=row["词牌"])

if __name__ == "__main__":
    driver = GraphDatabase.driver(URI, auth=AUTH)
    import_cipai_csv(driver, CSV_PATH)
    driver.close()
```

**执行前：** 确认 Neo4j 已启动；可先用 `df.head(5)` 小批量测试。

### 3.4 Word 文档抽取（`scripts/extract_docx.py`）

从《宋词.docx》中抽取乐器、流派、朝代等关系，示例逻辑：

```python
from docx import Document
from neo4j import GraphDatabase

# 预定义映射表（根据文档内容整理）
INSTRUMENT_RELATIONS = [
    ("慢词", "笙"),
    ("小令", "琵琶"),
]
STYLE_POETS = [
    ("清雅派", "姜夔", "南宋"),
    ("豪放派", "苏轼", "北宋"),
]

def import_from_docx(driver):
    with driver.session() as session:
        for ci_type, instrument in INSTRUMENT_RELATIONS:
            session.run("""
                MERGE (i:Instrument {name: $instrument})
                WITH i
                MATCH (c:Cipai {type: $ci_type})
                MERGE (c)-[:ACCOMPANIED_BY]->(i)
            """, ci_type=ci_type, instrument=instrument)

        for style, poet, period in STYLE_POETS:
            session.run("""
                MERGE (s:CiStyle {name: $style})
                MERGE (p:Poet {name: $poet})
                MERGE (per:Period {name: $period})
                MERGE (p)-[:BELONGS_TO]->(s)
                MERGE (p)-[:ACTIVE_IN]->(per)
            """, style=style, poet=poet, period=period)
```

若文档结构复杂，可先用正则或关键词匹配提取，再人工校对映射表。

### 3.5 可选：OWL 导入 Neo4j（n10s）

1. 在 Neo4j 中安装 [Neosemantics (n10s)](https://neo4j.com/labs/neosemantics/) 插件
2. 执行：

```cypher

CALL n10s.rdf.import.fetch("file:///path/to/SongCi_Ontology.owl", "RDF/XML");
```

概念层与实例层可共存：OWL 导入的类作为 Schema 参考，CSV/Word 导入的节点作为业务数据。

### 3.6 数据校验 Cypher

导入完成后在 Neo4j Browser 中执行：


### 4.2 后端 API 设计

**基础路径：** `http://localhost:8000/api`

| 接口 | 方法 | 参数 | 返回 | 说明 |
|------|------|------|------|------|
| `/graph` | GET | `period`, `limit` | `{ nodes, links }` | 全图或按朝代过滤 |
| `/search` | GET | `q`, `type` | `{ nodes, links }` | 按词人/词牌/乐器搜索 |
| `/node/{id}` | GET | — | `{ id, label, properties, related }` | 节点详情（侧边栏） |
| `/timeline` | GET | — | `[{ period, count }]` | 各时期节点数量 |

**`/graph` 示例 Cypher：**

```cypher
MATCH (p:Poet)-[r1:REPRESENT]->(c:Cipai)-[r2:ACCOMPANIED_BY]->(i:Instrument)
WHERE $period IS NULL OR (p)-[:ACTIVE_IN]->(:Period {name: $period})
RETURN p, c, i, r1, r2
LIMIT $limit
```

**FastAPI 路由示例（`backend/routers/graph.py`）：**

```python
from fastapi import APIRouter, Query
from neo4j import GraphDatabase

router = APIRouter(prefix="/api")
driver = GraphDatabase.driver("bolt://localhost:7687", auth=("neo4j", "pwd"))

@router.get("/graph")
def get_graph(period: str = None, limit: int = 100):
    query = """
    MATCH (p:Poet)-[:REPRESENT]->(c:Cipai)
    OPTIONAL MATCH (c)-[:ACCOMPANIED_BY]->(i:Instrument)
    WHERE $period IS NULL OR EXISTS {
        MATCH (p)-[:ACTIVE_IN]->(:Period {name: $period})
    }
    RETURN p, c, i LIMIT $limit
    """
    with driver.session() as session:
        result = session.run(query, period=period, limit=limit)
        nodes, links = [], []
        seen = set()
        for record in result:
            for key in ("p", "c", "i"):
                node = record[key]
                if node and node.element_id not in seen:
                    seen.add(node.element_id)
                    nodes.append({
                        "id": node.element_id,
                        "label": node.get("name"),
                        "group": list(node.labels)[0]
                    })
            # 根据 record 构建 links ...
        return {"nodes": nodes, "links": links}
```

## 八、常见问题

**Q：CSV 列名与文档不一致怎么办？**  
先用 `pandas.read_csv` 打印 `df.columns`，建立列名映射字典后再写入 Cypher。

**Q：Neo4j 与 Protégé 的类名要不要完全一致？**  
概念层保持一致即可；Neo4j 节点标签可用 `:Poet` 对应 OWL 中的 `Poet`，不必带命名空间前缀。

**Q：3d-force-graph 节点太多卡顿？**  
接口默认 `limit=100`；按朝代过滤减少节点数；或改用 2D 力导向图作为备选。

**Q：Word 文档无法自动抽取？**  
先用人工整理映射表（如本文 3.4 节示例），保证答辩数据准确；后续再考虑 NLP 抽取。

**Q：没有服务器怎么部署？**  
答辩时使用本地三套工具切换即可；可提前录制 Web 操作视频作为备份。
